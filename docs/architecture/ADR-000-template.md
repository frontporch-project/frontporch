## Date

## Context

Describe the situation, constraint, or tradeoff that requires a decision.

Keep this short. Include only the background needed to understand the decision.

## Decision

State the decision directly.

Use clear project language. Avoid implementation detail unless it is central to the decision.

## Consequences

Describe the important outcomes of this decision.

Include benefits, costs, constraints, and risks. Be honest about tradeoffs.

## Future Considerations

Document conditions that may cause this decision to be revisited.

If this ADR is superseded later, link to the newer ADR.
